## 課程說明

本課程涵蓋資料探勘的核心理論與實作技術，從資料前處理、探索式分析，到分類、分群、關聯規則、迴歸等經典演算法，搭配 Python 生態系工具進行全程實作。課程後半導入 AI 輔助開發流程，培養學生運用 AI 工具加速資料分析與模型建構的能力。

### 課程核心主題

**Python 資料科學基礎**：使用 Anaconda 建立 Python 開發環境，熟悉 Pandas、NumPy、Matplotlib 等資料科學核心套件，掌握資料讀取、清洗、轉換與視覺化的完整流程。

**資料前處理與特徵工程**：學習缺失值處理、異常值偵測、資料正規化、特徵選擇與特徵萃取等前處理技術，理解高品質資料對模型效能的關鍵影響。

**監督式學習**：實作決策樹、隨機森林、K 近鄰演算法、單純貝氏分類器、支援向量機、邏輯迴歸與線性迴歸等演算法，學習模型訓練、評估與調校的完整流程。

**非監督式學習**：實作 K-Means 分群、階層式分群與 DBSCAN 密度分群演算法，搭配輪廓係數等指標評估分群品質。學習 Apriori 關聯規則探勘，從交易資料中發掘有意義的規則。

**模型評估與優化**：掌握混淆矩陣、精確率、召回率、F1 分數、ROC 曲線、交叉驗證等評估方法，學習超參數調校與過擬合防範策略。

**AI 工具輔助開發**：從第 10 週起導入 AI 工具於資料分析與模型開發流程中。學習 Prompt Engineering 技巧，掌握如何對 ChatGPT、Copilot 等 AI 工具下達有效指令，輔助資料清洗、程式碼產生、模型選擇與結果解讀，同時培養驗證 AI 產出正確性的能力。核心原則為「先理解再使用」，確保學生具備獨立判斷與修正 AI 產出的能力。

## 作業繳交方式

本課程採用 **Fork + Pull Request** 流程繳交作業：

1. Fork 本 Repo 到你的 GitHub 帳號

3. 發 Pull Request 回本 Repo 繳交

每週作業放在對應的資料夾中，例如 `week01/`、`week02/`。

## 期末專題

### 專題說明

本課程包含分組期末專題，要求運用資料探勘技術，針對一份與**海事或海洋**相關的資料集進行完整的探勘分析，並產出分析報告。

### 專題題目方向參考

以下為建議的專題題目方向，各組可從中選擇或自行提出相關主題：

**方向一：船舶交通與航運分析**
- 利用 AIS 船舶自動識別系統資料，分析港口船舶進出模式與航線分群
- 航運貨物量預測：根據歷史數據預測特定港口未來貨物吞吐量
- 船舶類型分類：依據船舶特徵資料進行自動分類
- 航運事故風險因子分析：探索影響海事事故的關鍵因素

**方向二：海洋環境與生態**
- 海水溫度與鹽度變化趨勢分析與預測
- 海洋廢棄物熱點分群：利用淨灘資料分析廢棄物分布模式
- 珊瑚礁白化風險因子探勘：結合水溫、酸鹼值等環境因子
- 海洋生物多樣性分析：利用調查資料進行物種分布模式探勘

**方向三：漁業資料探勘**
- 漁獲量預測：依據季節、海溫、洋流等因子預測漁獲量
- 漁場分群分析：利用漁業資料識別高產漁場的空間分布
- 漁船作業模式分類：根據軌跡資料辨識不同作業型態
- 水產養殖水質異常偵測：利用感測器資料建立預警模型

**方向四：港口營運與物流**
- 港口貨櫃裝卸效率分析與瓶頸識別
- 船舶到港時間預測：結合氣象、航線、船速等因子
- 港口能源消耗模式分群與節能潛力分析
- 關聯規則探勘：分析貨物種類與航線的關聯性

**方向五：氣象海象與災害預警**
- 颱風路徑分群與影響範圍分析
- 波浪高度預測：結合風速、氣壓等因子的迴歸模型
- 海岸侵蝕風險因子探勘
- 海上搜救最佳部署位置分析

## Git 基本流程

### 首次設定（只需做一次）

安裝 Git 後，設定你的姓名和 Email：

```bash
git config --global user.name "你的姓名"
git config --global user.email "你的Email"
```

### Fork + Clone（每學期做一次）

1. 在老師的 Repo 頁面右上角點選 **Fork**，將 Repo 複製到你的帳號下
2. Clone 你的 Fork 到本機：

```bash
git clone https://github.com/你的帳號/114-2_DataMining.git
cd 114-2_DataMining
```

### 每週作業流程

```
建立作業資料夾 → 編輯檔案 → add → commit → push → 發 PR
```

Step 1：建立當週作業資料夾並完成作業

```bash
mkdir week01
cd week01
# 建立作業檔案，完成作業內容
```

Step 2：將作業加入版本控制並推送

```bash
git add .
git commit -m "完成第1週作業"
git push origin main
```

Step 3：到 GitHub 網頁發 Pull Request

1. 進入你的 Fork 頁面
2. 點選 **Contribute** > **Open pull request**
3. 確認內容後點選 **Create pull request**
4. PR 標題格式：`學號_姓名_week01`

### 同步老師的最新版本

當老師更新了課程 Repo，你的 Fork 不會自動更新。請依照以下步驟取得最新版本：

1. 進入你的 Fork 頁面：`https://github.com/你的帳號/114-2_DataMining`
2. 頁面上方會出現提示：`This branch is X commits behind pychang-ai:main`
3. 點選「**Sync fork**」按鈕
4. 點選「**Update branch**」
5. 回到本機，執行：
   ```bash
   cd 114-2_DataMining
   git pull origin main
   ```

這樣你的 Fork 和本機都會更新到最新版本。你自己建立的作業資料夾不受影響。

| `git clone <url>` | 複製遠端 Repo 到本機 | `git clone https://github.com/user/repo.git` |
| `git status` | 查看目前檔案狀態 | `git status` |
| `git add <file>` | 將檔案加入暫存區 | `git add week01/q1_env.txt` |
| `git add .` | 將所有變更加入暫存區 | `git add .` |
| `git commit -m "訊息"` | 提交變更並附上說明 | `git commit -m "完成第1週作業"` |
| `git push origin main` | 將本機提交推送到 GitHub | `git push origin main` |
| `git pull origin main` | 從 GitHub 拉取最新版本 | `git pull origin main` |
| `git log` | 查看提交歷史紀錄 | `git log --oneline` |

### Fork 與 Pull Request 指令

| 指令 | 功能 | 使用範例 |
|------|------|---------|
| Fork | 在 GitHub 網頁上操作，複製老師的 Repo 到你的帳號 | 點選 Repo 右上角 Fork 按鈕 |
| Pull Request | 在 GitHub 網頁上操作，將你的作業提交回老師的 Repo | 點選 Contribute > Open pull request |

### 常用輔助指令

| 指令 | 功能 | 使用範例 |
|------|------|---------|
| `git diff` | 查看檔案變更的內容 | `git diff` |
| `git log --oneline` | 用精簡格式查看歷史紀錄 | `git log --oneline -5` |
| `git remote -v` | 查看遠端 Repo 的連結 | `git remote -v` |
| `git branch` | 查看目前所在的分支 | `git branch` |
